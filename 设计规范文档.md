## 一、设计原则与约束

### 1.1 核心定位
**数据查询型小程序**，非资讯网站，非社区平台。

### 1.2 设计原则

| 原则 | 说明 | 落地要求 |
|------|------|----------|
| 数据优先 | 比分、排名、状态信息优先级最高 | 首屏必须展示比赛数据 |
| 阅读效率 | 信息层级清晰，一扫即懂 | 避免复杂装饰，留白充足 |
| 轻量体验 | 符合微信小程序使用习惯 | 使用原生导航、原生TabBar |
| 稳定可用 | 考虑API限制和异常情况 | 所有状态必须有兜底设计 |

### 1.3 技术约束清单

| 约束项 | 设计应对 |
|--------|----------|
| 导航栏必须使用原生 | `navigationStyle: default`，背景色#38003C |
| TabBar必须使用原生 | 4个入口：首页/积分榜/赛程/我的 |
| 10次/分钟API限流 | 显示"数据来自缓存"标识 |
| 15-30分钟数据延迟 | 显示"更新于X分钟前" |
| 图片可能加载失败 | 所有图片必须有默认占位图 |
| 休赛期无数据 | 空状态组件必须完善 |

### 1.4 模块优先级

**核心模块（P0）**：
- 比赛卡片（首页、赛程）
- 积分榜列表
- 球队详情信息
- 关注功能

**次要模块（P1）**：
- 搜索功能
- 球员详情
- 战绩展示

**弱化模块（P2）**：
- 资讯模块（首页仅预留位）
- 复杂设置项

### 2.1 主色调

| 名称 | 色值 | 用途 |
|------|------|------|

| 名称 | 色值 | 用途 |

## 三、字体与排版

### 3.1 字体规范

微信小程序默认使用系统字体：
- iOS: PingFang SC
- Android: Noto Sans CJK

### 3.2 字号规范

| 层级 | 字号 | 字重 | 行高 | 用途 |
|------|------|------|------|------|
| **导航栏标题** | 36rpx | bold | 44rpx | 页面标题 |
| **比分数字** | 48rpx | bold | 56rpx | 比赛比分 |
| **排名数字** | 32rpx | bold | 40rpx | 积分榜排名 |
| **卡片标题** | 32rpx | bold | 40rpx | 卡片标题 |
| **正文内容** | 28rpx | normal | 40rpx | 列表项、描述 |
| **辅助文字** | 24rpx | normal | 32rpx | 时间、状态 |
| **标签文字** | 20rpx | normal | 28rpx | 状态标签、角标 |

### 3.3 字体颜色使用

| 场景 | 颜色 |
|------|------|
| 主要标题/球队名 | `#333333` |
| 次要信息/时间 | `#666666` |
| 提示/禁用 | `#999999` |
| 高亮/实时 | `#00FF85` |
| 链接/可点击 | `#38003C` |

## 四、间距与布局

### 4.1 页面布局

```
┌─────────────────────────────┐  ← 系统状态栏 (iOS 44px / Android 24dp)
│ 原生导航栏 (44px)            │  ← #38003C 背景
├─────────────────────────────┤
│                             │
│      页面内容区域            │  ← 背景 #F5F5F5
│      (左右边距 30rpx)        │
│                             │
├─────────────────────────────┤
│ 原生TabBar (50px)            │  ← 背景 #FFFFFF / 选中 #38003C
└─────────────────────────────┘
```

### 4.2 间距规范

| 名称 | 数值 | 用途 |
|------|------|------|
| **页面边距** | 30rpx | 左右边距 |
| **卡片间距** | 20rpx | 卡片之间 |
| **卡片内边距** | 24rpx | 卡片内部 |
| **列表项高度** | 100rpx | 球队列表项 |
| **列表项间距** | 0 | 列表项之间用分割线 |
| **元素间距** | 16rpx | 同行元素之间 |
| **文字间距** | 8rpx | 文字与图标之间 |

### 4.3 圆角规范

| 名称 | 数值 | 用途 |
|------|------|------|
| **大圆角** | 16rpx | 焦点卡片 |
| **标准圆角** | 12rpx | 普通卡片 |
| **小圆角** | 8rpx | 按钮、标签 |
| **全圆角** | 50% | 头像、队徽 |

### 4.4 尺寸规范

| 元素 | 尺寸 |
|------|------|
| **队徽** | 60rpx × 60rpx |
| **球员头像** | 80rpx × 80rpx |

|------|----------|----------|
| 积分榜 | 5分钟 | "数据来自缓存" |
| 赛程 | 1分钟 | "更新于X分钟前" |
| 球队列表 | 1小时 | - |
| 球队详情 | 10分钟 | "数据来自缓存" |
| 球员详情 | 10分钟 | "数据来自缓存" |

### 8.6 异常处理规范

| 异常情况 | 处理方式 | UI展示 |
|----------|----------|--------|
| 网络断开 | 使用缓存数据，显示提示 | error-state组件 |
| 接口限流(429) | 使用缓存数据，提示稍后刷新 | "请求过于频繁，请稍后再试" |
| 数据为空 | 显示空状态 | empty-state组件 |
| 图片加载失败 | 显示默认占位图 | default/team.png |
| 接口超时 | 重试3次后显示错误 | error-state组件 |
