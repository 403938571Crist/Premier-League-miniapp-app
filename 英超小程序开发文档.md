# 英超资讯小程序 - 开发文档

## 1. 项目概述

### 1.1 项目名称
英超资讯微信小程序

### 1.2 项目目标
面向英超球迷提供轻量、快速、稳定的数据查询服务，核心功能包括：
- 查看英超积分榜
- 查看比赛赛程与比分状态
- 查看球队详情
- 查看球员详情
- 管理关注球队

本项目定位为“数据查询型微信小程序”，不是资讯网站、社区或会员平台。首页可保留少量资讯模块，但不作为主功能。

### 1.3 MVP 范围
本期必须交付：
- 首页
- 积分榜
- 赛程
- 球队详情
- 球员详情
- 我的

本期不做：
- 评论系统
- 用户社交
- 直播视频
- 购票
- 会员等级体系
- 商城

---

## 2. 技术方案

### 2.1 技术栈
- 平台：微信小程序
- 前端：原生小程序
- 语言：JavaScript
- 视图：WXML
- 样式：WXSS
- 网络请求：`wx.request`
- 本地缓存：`wx.storage`

### 2.2 数据源
- 第三方 API：`football-data.org`
- 主要赛事：英超 `PL`

### 2.3 技术约束
- 免费版接口有限流：10 次/分钟
- 数据可能存在 15-30 分钟延迟
- 休赛期部分接口可能返回空数据
- 球员资料、头像、队徽可能不完整

因此开发必须包含：
- 接口缓存
- 加载状态
- 空状态
- 错误状态
- 图片失败兜底

## 4. 页面开发说明

### 4.1 首页 `pages/index/`

#### 功能目标
展示焦点比赛、今日比赛、快捷入口和少量资讯入口。

#### 页面模块
- 顶部标题区/搜索区
- 焦点比赛卡片
- 快捷入口
- 今日比赛列表
- 资讯预留模块（可选）

#### 必要功能
- 支持跳转积分榜页
- 支持跳转赛程页
- 支持跳转球队详情页
- 支持跳转球员详情页
- 展示比赛状态：未开始 / 进行中 / 已结束

#### 数据来源
- 焦点比赛、今日比赛：`GET /v4/competitions/PL/matches`

#### 开发说明
- 首页首屏优先展示比赛，不要让资讯卡片抢主功能
- 焦点比赛可从当天或最近比赛中筛选 1-3 场
- 若当日无比赛，显示“今日暂无比赛”

### 4.2 积分榜 `pages/standings/`

#### 功能目标
快速展示英超排名和球队战绩。

#### 页面模块
- 顶部赛季信息
- Tab：总榜 / 主场 / 客场
- 排名列表

#### 核心字段
- 排名
- 队徽
- 球队名
- 已赛
- 净胜球
- 积分

#### 扩展字段
- 胜 / 平 / 负
- 进球 / 失球

#### 数据来源
- `GET /v4/competitions/PL/standings`

#### 开发说明
- 移动端优先采用竖向列表，不建议做重型横向滚动大表格
- 前四和降级区可做轻量标识
- 点击球队进入球队详情页

### 4.3 赛程 `pages/fixtures/`

#### 功能目标
按日期查看比赛安排和比分。

#### 页面模块
- 顶部标题区
- 日期切换区
- 比赛分类 Tab：全部 / 进行中 / 结果
- 比赛列表

#### 核心字段
- 日期
- 轮次
- 主队 / 客队
- 队徽
- 比分
- 状态
- 比赛时间

#### 数据来源
- `GET /v4/competitions/PL/matches`

#### 开发说明
- 支持左右切换日期
- 支持筛选不同比赛状态
- 卡片内保留“比赛详情”预留入口，但 MVP 可先不做完整比赛详情页

### 4.4 球队详情 `pages/team-detail/`

#### 功能目标
展示球队基础资料、阵容和近期赛程。

#### 页面模块
- 头部球队信息区
- 关注按钮
- Tab：概览 / 阵容 / 赛程

#### 头部字段
- 队徽
- 中文名
- 英文名
- 主场
- 教练
- 当前排名或积分

#### 概览字段
- 近期战绩
- 赛季统计
- 简要介绍

#### 阵容字段
- 球员姓名
- 位置
- 国籍
- 球衣号码

#### 数据来源
- 球队详情：`GET /v4/teams/{id}`
- 阵容：球队详情接口返回的 `squad`
- 球队赛程：从比赛接口按球队 ID 过滤

#### 开发说明
- “关注”状态本地存储即可
- 阵容页按门将、后卫、中场、前锋分组展示

### 4.5 球员详情 `pages/player-detail/`

#### 功能目标
展示球员资料和赛季基础数据。

#### 页面模块
- 头像与身份信息
- 基本资料
- 赛季数据

#### 核心字段
- 姓名
- 位置
- 国籍
- 出生日期
- 身高
- 球衣号码
- 所属球队
- 出场
- 进球
- 助攻

#### 数据来源
- `GET /v4/persons/{id}`

#### 开发说明
- 如果第三方 API 不返回完整赛季数据，页面需支持缺失字段隐藏或显示 `--`
- 头像缺失时显示默认占位

### 4.6 我的 `pages/profile/`

#### 功能目标
管理用户偏好和设置。

#### 页面模块
- 用户信息区
- 我的关注
- 设置区

#### 设置项
- 清除缓存
- 关于我们
- 意见反馈

#### 数据来源
- 本地存储

#### 开发说明
- 不做会员体系
- 不做积分、勋章、购票等扩展功能
- 我的关注至少支持球队列表与取消关注

## 6. API 设计

### 6.1 请求基础配置
- Base URL：`https://api.football-data.org/v4`
- Header：
  - `X-Auth-Token: {API_KEY}`

### 6.2 接口清单

#### 6.2.1 获取积分榜
- 接口：`GET /competitions/PL/standings`
- 用途：积分榜页面
- 缓存：5 分钟

#### 6.2.2 获取比赛列表
- 接口：`GET /competitions/PL/matches`
- 用途：首页、赛程页、球队赛程
- 缓存：1 分钟

#### 6.2.3 获取球队列表
- 接口：`GET /competitions/PL/teams`
- 用途：球队信息补全、搜索
- 缓存：1 小时

#### 6.2.4 获取球队详情
- 接口：`GET /teams/{id}`
- 用途：球队详情页
- 缓存：10 分钟

#### 6.2.5 获取球员详情
- 接口：`GET /persons/{id}`
- 用途：球员详情页
- 缓存：10 分钟

## 7. 数据处理与缓存

### 7.1 缓存策略
- 积分榜：5 分钟
- 比赛列表：1 分钟
- 球队列表：1 小时
- 球队详情：10 分钟
- 球员详情：10 分钟

### 7.2 缓存实现建议
- 本地结构：
  - `key`
  - `timestamp`
  - `data`

### 7.3 请求逻辑
1. 先查本地缓存
2. 缓存有效则直接返回
3. 缓存失效则请求接口
4. 请求成功后写入缓存
5. 请求失败则尝试回退旧缓存

### 7.4 本地存储项
- `favoriteTeams`
- `userProfile`
- `cache:standings`
- `cache:matches`
- `cache:teams`

---

## 8. 状态处理

### 8.1 通用状态
每个页面必须具备以下状态：
- 加载中
- 无数据
- 请求失败
- 图片失败

### 8.2 页面级状态要求
- 首页：无比赛时显示空状态
- 积分榜：接口失败时显示错误提示与重试按钮
- 赛程：无比赛时显示“该日期暂无比赛”
- 球队详情：无阵容时显示“暂无阵容数据”
- 球员详情：字段缺失显示 `--`

## 13. 风险与注意事项

### 13.1 数据风险
- 第三方 API 限流可能影响体验
- 球员数据完整性不稳定
- 非比赛日数据更新价值较低

### 13.2 开发风险
- 若首页资讯内容过重，会偏离产品定位
- 若积分榜采用过宽表格，移动端体验差
- 若不做缓存，接口限流会快速触发

### 13.3 建议
- 先完成“比赛 + 积分榜 + 球队/球员详情”闭环
- 资讯模块先预留，不作为主开发重点
- 确保 UI 设计服务于开发效率，不做高成本视觉效果

---

## 14. 一句话交付结论
本项目应按“英超数据查询型微信小程序”进行开发，优先完成积分榜、赛程、球队详情、球员详情和关注能力，首页只承担导流与信息汇总作用，不扩展为资讯媒体或会员产品。
